# demo/AGENTS.md

このファイルはdemoディレクトリ向けのAIエージェント指示書です。

## 必須：ライブラリを使うこと

### MIDIパース

**SMFバイナリのパースには必ずnpmライブラリを使うこと。自前実装禁止。**

- ❌ 禁止：`MidiReader` のようなカスタムバイナリリーダーを自前実装する
- ❌ 禁止：`parseMidiNotes.ts` のような独自のSMFパーサーを自前実装する
- ✅ 推奨：`@tonejs/midi` などの確立されたMIDIライブラリを使う

自前でバイナリパーサーを実装すると、VLQ読み取り・ランニングステータス・テンポ変更など
SMF仕様の細かいエッジケースでバグが発生しやすく、デバッグが困難になる。

### オーディオ合成・再生

**オーディオ合成・再生にはTone.jsを使うこと。独自のサイン波合成等を実装禁止。**

- ❌ 禁止：`AudioContext` + 手動サイン波計算でオーディオを合成する
- ✅ 推奨：`Tone.Synth`、`Tone.PolySynth` などTone.jsのシンセサイザーを使う
- ✅ 推奨：Tone.jsの `Player`、`Waveform`、`FFT` を使って再生・可視化する

### WAVエクスポート

**WAVファイル出力にはライブラリを使うこと。自前でRIFFヘッダーを組み立て禁止。**

- ❌ 禁止：`DataView` で手動でWAVヘッダーを書くような自前実装
- ✅ 推奨：`audiobuffer-to-wav` 等の既存ライブラリを使う

## 利用可能なライブラリ（package.json）

```json
{
  "dependencies": {
    "web-tree-sitter": "^0.26.5",
    "tone": "^15.0.4"
  }
}
```

追加ライブラリが必要な場合は `npm install` で追加してよい。

## 一般原則

- 500行を超えるTypeScriptファイルは単一責任の原則に従って分割すること
- ライブラリが提供する機能を自前実装してはならない
- 自前実装はバグの温床となり、issue #89・#91のような根が深いトラブルの原因になる
